Hjälp & Begreppsförklaring

Vad är Modbus?

Modbus är ett kommunikationsprotokoll som används i industriella system för att läsa och skriva data från enheter som PLC:er, sensorer och mätare. Modbus TCP är varianten som kör över nätverk (Ethernet).

Register-typer

TypAdressområdeBeskrivning
Coil0xxxxLäs/skriv barriär - ON/OFF (1 bit)
Discrete Input1xxxxSkrivskyddad barriär - ON/OFF (1 bit)
Input Register3xxxxSkrivskyddat register (16 bitar)
Holding Register4xxxxLäs/skriv register (16 bitar) - vanligast!

Holding Register är den vanligaste typen för att läsa analoga värden som temperatur, flöde, tryck etc.

Byte Order (Endianness) - Vilken är vilken?

Datorer kan lagra bytes i olika ordning. Tänk dig att du har siffran 1000 (hex: 0x03E8).

I minnet lagras detta i 2 bytes. Ordningen beror på systemet:

InställningByte-ordningExempelAnvänds av
Big EndianMSB först03 E8De flesta PLC:er, Modbus-standarden
Little EndianLSB förstE8 03Intel/AMD-processorer, vissa PLC:er
Big Endian SwapMSB först, byt platsE8 03Vissa äldre system
Little Endian SwapLSB först, byt plats03 E8Sällsynt

Tips: Om du får konstiga värden, prova att byta byte order! Vanligast är Big Endian.

Word Order

För 32-bitars värden (float32, int32, uint32) behövs 2 register (words). Ordningen dessa två register kommer i kan variera:

InställningBeskrivning
BigHög word först (vanligast)
LittleLåg word först

Exempel: Värdet 1234.5 som float32 = register [0x44 0x9A 0x48 0x00]

Big Word Order: Register 1 = 0x449A, Register 2 = 0x4800

Little Word Order: Register 1 = 0x4800, Register 2 = 0x449A

Scaling Factor & Offset

Används för att omvandla råvärdet till verkligt värde:

Slutvärde = (Råvärde × Scaling Factor) + Offset

Exempel: Råvärde = 452, Scaling = 0.1, Offset = 0 → Resultat = 45.2 L/min

Vad är OPC UA?

OPC UA (Open Platform Communications Unified Architecture) är ett modernt industriellt kommunikationsprotokoll. Jämfört med Modbus:

Modbus TCPOPC UA
EnkelhetEnkelt, väletableratMer komplext, modernare
SäkerhetIngen inbyggdKryptering & autentisering
Data-identifieringRegister-adress (nummer)NodeId (t.ex. ns=1;s=Sensor1)
DataupptäcktManuell (du måste veta adressen)Kan bläddra i serverns nod-träd

OPC UA NodeId

En NodeId identifierar en specifik datapunkt i en OPC UA-server. Format:

ns=<namnrymd>;s=<strängnamn> eller ns=<namnrymd>;i=<heltal>

Exempel: ns=1;s=FlowSensor01 = Namnrymd 1, strängidentifierare "FlowSensor01"

Vanliga problem & lösningar

"Får konstiga värden" → Kontrollera byte order och word order. Prova Big Endian först.

"Kan inte ansluta" → Kontrollera IP-adress, port, och att brandvägg tillåter trafik. Modbus = port 502, OPC UA = port 4840.

"Timeout" → Enheten svarar inte. Kontrollera att den är påslagen och nätverkskabeln är ansluten.

"Värdet är alltid 0" → Kontrollera register-adress och register-typ. Många system börjar räkna från 0 eller 1 beroende på konvention.
